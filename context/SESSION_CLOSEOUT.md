# SESSION_CLOSEOUT.md

Use this at the end of a meaningful work session.

## Closeout Checklist

1. Update memory with stable project facts only.
2. Append real decisions to `context/DECISIONS.md`.
3. Update the owning file for changed content:
   - `guidelines.md` for official challenge brief details
   - future concept files for solution direction and prototype evolution
   - other project files as needed
4. Record the next active focus.
5. Do not store detailed specs in memory if a project file should own them.

- Decisions to log:
- Files to update:
